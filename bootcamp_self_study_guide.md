# Self-Directed Software Development Bootcamp
## 36-Week Comprehensive Study Plan

---

## ðŸ“‹ Overview
**Total Duration:** 36 weeks  
**Weekly Commitment:** 15-17.5 hours
- 5 hours: Video tutorials & lectures
- 2.5 hours: Practice labs & projects (bi-weekly)
- 10 hours: Independent coding & building

**Your Advantage:** Using Claude + Cursor for AI-assisted learning
- Focus on **understanding concepts** rather than memorizing syntax
- Use AI to explain code, debug, and explore alternative approaches
- Learn to prompt effectively for better code assistance

---

## ðŸŽ¯ Learning Philosophy

### How to Study Effectively:
1. **Concept First, Code Second**
   - Understand WHY before HOW
   - Draw diagrams, explain concepts in your own words
   - Ask Claude to explain concepts at different levels

2. **Build, Break, Fix**
   - Don't just follow tutorials - experiment
   - Intentionally break code to understand error messages
   - Use Cursor to refactor and improve your code

3. **Industry Mindset**
   - Think about scalability, maintainability, security
   - Learn version control from day 1 (Git)
   - Write README files for all projects
   - Comment your code thoughtfully

4. **Active Learning with AI**
   - Ask Claude "Why does this work?" not just "Make this work"
   - Request explanations of industry best practices
   - Use AI for code reviews and optimization suggestions

## ðŸ’¡ Tips for Success with Claude + Cursor

### How to Use AI Effectively:

#### For Learning:
1. **Ask "Why" questions:**
   - "Why is async/await better than callbacks?"
   - "Why do we need virtual DOM in React?"
   - "Why is this pattern considered best practice?"

2. **Request explanations at different levels:**
   - "Explain this like I'm 5"
   - "Explain this at intermediate level"
   - "Explain the internals"

3. **Ask for comparisons:**
   - "Compare useState vs useReducer"
   - "When should I use SQL vs NoSQL?"
   - "REST API vs GraphQL - pros and cons"

#### For Building:
1. **Start with pseudocode:**
   - Outline logic before coding
   - Let AI help structure your thoughts
   - Ask for architecture advice

2. **Request code reviews:**
   - "Review this code for best practices"
   - "How can I optimize this?"
   - "What security issues do you see?"

3. **Debugging:**
   - Share error messages
   - Explain expected vs actual behavior
   - Ask for debugging strategies

4. **Refactoring:**
   - "Make this more maintainable"
   - "Improve performance of this function"
   - "Make this code more DRY"

#### DON'T:
- Copy code without understanding it
- Skip fundamentals because AI can do it
- Rely on AI for every small thing
- Forget to test and debug yourself

#### DO:
- Use AI as a tutor and pair programmer
- Ask for explanations of generated code
- Challenge AI suggestions
- Verify information from multiple sources
- Build independently sometimes to test understanding

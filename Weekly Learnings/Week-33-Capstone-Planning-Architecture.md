# Week 33 — Capstone: Planning & Architecture

**Theme:** Capstone Project  
**Suggested time:** 12–16 hours total

---

## Learning goals

- Define a real problem and target users
- User stories and acceptance criteria
- Wireframes and simple mockups (Figma)
- System architecture (frontend, API, DB, auth)
- Tech stack and database schema
- Project repo structure and roadmap (e.g. GitHub Projects)

---

## Activities

- [ ] Pick problem domain (SaaS, social, e-commerce, data, productivity)
- [ ] Research existing solutions; note differentiators
- [ ] User personas and 5–10 user stories with acceptance criteria
- [ ] Wireframes: main flows (e.g. signup, core action, settings)
- [ ] Architecture diagram: client, API, DB, auth, optional storage/queue
- [ ] Choose stack (e.g. React, Node/Express, MongoDB; or variant)
- [ ] Database schema: collections/tables, relations, indexes
- [ ] Create repo; README with overview; GitHub Project with tasks for Weeks 34–36

---

## Deliverables

- [ ] Problem statement (1 page)
- [ ] User stories + acceptance criteria
- [ ] Wireframes (Figma or similar)
- [ ] System architecture diagram
- [ ] Database schema (document or diagram)
- [ ] Project board (backlog for next 3 weeks)
- [ ] README: project overview, stack, how to run (placeholder OK)

---

## Weekly time split (12–16 hrs)

| Activity | Hours |
|----------|--------|
| Problem + research + user stories | 4–5 |
| Wireframes + architecture + schema | 4–5 |
| Repo + board + README | 3–4 |
| Buffer / deeper design | 1–2 |

---

## Checklist

- [ ] Clear problem and scope for capstone
- [ ] Architecture and schema documented
- [ ] Ready to code (Week 34)

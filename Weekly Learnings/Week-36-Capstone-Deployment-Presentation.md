# Week 36 — Capstone: Deployment & Presentation

**Theme:** Capstone Project  
**Suggested time:** 12–16 hours total

---

## Learning goals

- Deploy capstone (frontend + backend + DB)
- Custom domain and SSL (optional)
- CI/CD for capstone repo
- Monitoring or error tracking (e.g. Sentry)
- Documentation: README, architecture, API, user guide
- Demo video (5 min) and presentation
- Practice pitch for interviews

---

## Activities

- [ ] Deploy backend and frontend (e.g. Render + Vercel; or Docker on one platform)
- [ ] Configure env in production; verify DB and auth
- [ ] Optional: custom domain, SSL
- [ ] GitHub Actions: test on push; deploy on merge to main
- [ ] Add error tracking (e.g. Sentry); fix critical issues
- [ ] Write README: problem, features, stack, setup, deploy link
- [ ] Document architecture (diagram + short description)
- [ ] API docs: endpoints, auth, example requests (README or Postman)
- [ ] Short user guide (how to sign up and use main flow)
- [ ] Record 5-minute demo video (screen + voice)
- [ ] Create presentation slides (problem, solution, tech, demo, learnings)
- [ ] Practice pitch 2–3 times; time yourself
- [ ] Optional: write a case study blog post

---

## Deliverables

- [ ] Live deployed application (URL in README)
- [ ] GitHub repo with README, architecture, API docs
- [ ] User guide (in README or separate doc)
- [ ] 5-minute demo video (link in README)
- [ ] Presentation slides (for interviews or portfolio)
- [ ] Optional: blog post or LinkedIn post about the project

---

## Weekly time split (12–16 hrs)

| Activity | Hours |
|----------|--------|
| Deployment + CI/CD + monitoring | 4–5 |
| Documentation (README, architecture, API, guide) | 3–4 |
| Demo video + slides | 3–4 |
| Practice pitch + buffer | 2–3 |

---

## Checklist

- [ ] Capstone is live and stable
- [ ] Documentation is complete and clear
- [ ] You can demo and explain the project in 5 minutes
- [ ] **Congratulations — you’ve completed the 36-week bootcamp.**

---

## What’s next

- Keep applying (5–10 applications per week)
- Keep doing 1–2 LeetCode (or equivalent) per day
- Add system design practice if targeting senior roles
- Iterate on resume and portfolio based on feedback
- Use your capstone as the centerpiece of “Tell me about a complex project”
